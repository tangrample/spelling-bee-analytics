📖 How to Use

  1. Play Spelling Bee: Go to nytimes.com/puzzles/spelling-bee
  2. Click the bookmarklet from your bookmarks bar
  3. Download automatically starts → saves spelling_bee_raw.json
  4. Move the file to your Spelling Bee analytics folder
  5. Run update script: ./update_spelling_bee.sh

⚠️ Why Daily Collection?

NYT only stores today + yesterday's data in your browser. To build historical records, you need to capture data daily before yesterday's puzzle disappears.

🔧 Manual Alternative

If the bookmarklet doesn't work, use the browser console method:

  1. Open NYT Spelling Bee
  2. Press F12 (or Cmd+Option+I on Mac) to open DevTools
  3. Go to the Console tab
  4. Paste this code:
const data = {}; for (let i = 0; i < localStorage.length; i++) { const key = localStorage.key(i); if (key.includes('spelling_bee') || key === '_window_gameData') { data[key] = JSON.parse(localStorage.getItem(key)); } } console.log(JSON.stringify(data, null, 2));

Then copy the output and save to spelling_bee_raw.json

📊 What Gets Captured

  • Your words found for today & yesterday
  • Your current rank (Genius, Amazing, etc.)
  • Total score and points
  • Answer key (only for revealed/yesterday's puzzles)
  • All puzzle metadata (letters, pangrams, etc.)

✅ Database Updates

After extraction, your analytics pipeline will:

  • Parse the raw data
  • Calculate achievements (GN4L, Queen Bee, etc.)
  • Update your SQLite database
  • Track historical trends
