🐝 Correct Data Extraction for NYT Spelling Bee

⚠️ IMPORTANT: Your previous extractions were missing _window_gameData which contains puzzle dates and metadata. Use this corrected method!

Step-by-Step Console Method

  1. Open NYT Spelling Bee: https://www.nytimes.com/puzzles/spelling-bee
  2. Wait for the game to FULLY load - let the puzzle appear and play at least one word
  3. Open DevTools: Press F12 (or Cmd+Option+I on Mac)
  4. Go to Console tab
  5. Paste this COMPLETE code:
// Extract complete Spelling Bee data
const data = {};

// Get localStorage data
for (let i = 0; i < localStorage.length; i++) {
  const key = localStorage.key(i);
  if (key.includes('spelling_bee')) {
    try {
      data[key] = JSON.parse(localStorage.getItem(key));
    } catch (e) {
      console.error('Error parsing localStorage key:', key);
    }
  }
}

// CRITICAL: Get window.gameData (contains puzzle info)
if (window.gameData) {
  data['_window_gameData'] = window.gameData;
  console.log('✓ Found window.gameData');
} else {
  console.error('✗ window.gameData NOT FOUND - game may not be loaded!');
  console.log('Make sure:');
  console.log('1. The puzzle has fully loaded');
  console.log('2. You have played at least one word');
  console.log('3. Try refreshing and waiting a few seconds');
}

// Output the complete data
console.log(JSON.stringify(data, null, 2));
  1. Check the output - You should see "_window_gameData" in the output
  2. Copy ONLY the JSON (starting with {, not console log messages)
  3. Save to: spelling_bee_raw.json

Common Issues

If window.gameData is missing:

  • Wait longer - Let the puzzle fully load (bee animation completes)
  • Play a word - Sometimes it only loads after interaction
  • Refresh the page - Then wait and try again
  • Check browser console - Look for any errors

If you copied console log prefix:

Remove lines like [Log] { (5155...js, line 2) - only keep the actual JSON starting with {
